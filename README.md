# Rescheduling Effort in Dynamic Flexible Job Shops — Realizable Value and the Applicability of Learning

Code and experiment workspace for the paper
**《响应时间约束下动态柔性作业车间重调度的力度决策：可实现价值上限与学习适用性的实证研究》**
(*Rescheduling effort in dynamic flexible job shops: realizable value and the applicability of learning*).

This is an empirical study. It abstracts the online rescheduling decision as the choice of a **rescheduling effort** along a monotone ladder — **L0** (minimal right-shift) ⊆ **L1** (small neighborhood) ⊆ **L2** (impact cluster) ⊆ **L3** (full reopen) — all sharing one CP-SAT exact-repair backend under a response budget, and quantifies its **realizable value** over the trivial minimal-right-shift baseline. The headline result is a **two-mechanism** finding: effort has structurally little leverage at low disturbance, while aggressive repair hits a **feasibility wall** at high disturbance — so no operating point with realizable value exists, and minimal right-shift is the robust default.

## Quickstart

```bash
pip install -r requirements.txt

# Frozen episodes + paper figures/summaries ship under outputs/. The bulky
# per-event grids are regenerated by the pipeline (they are not shipped):
python scripts/run_intensity_grid.py              # §7.2-7.4 recompute effort grid
python scripts/analyze_intensity_sensitivity.py   # §7.5 figures F1-F4, Tables 6/7
python scripts/run_rho_boundary_experiment.py     # §7.6 feasibility wall, Table 8, Figs 5/6
python -m unittest discover -s tests              # unit tests

# Or just inspect the shipped paper figures/tables directly:
#   outputs/sensitivity/  and  outputs/rho_boundary/
```

Requires Python ≥ 3.10 with `ortools` and `torch` (see [README_EN.md](README_EN.md#environment)).
